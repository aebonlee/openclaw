# 인증 및 데이터베이스

## Supabase Auth

### PKCE 플로우

Supabase Auth는 PKCE (Proof Key for Code Exchange) 플로우를 사용하여 보안을 강화한다.

```javascript
// supabase.js
import { createClient } from '@supabase/supabase-js';

export const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY,
  {
    auth: {
      flowType: 'pkce',
      autoRefreshToken: true,
      persistSession: true,
      detectSessionInUrl: true,
    },
  }
);
```

### OAuth Providers

| Provider | 설정 |
|----------|------|
| **Google** | Google Cloud Console에서 OAuth 2.0 클라이언트 ID 설정 |
| **Kakao** | Kakao Developers에서 REST API 키 설정 |

```javascript
// Google 로그인
const signInWithGoogle = async () => {
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'google',
    options: {
      redirectTo: `${window.location.origin}/`,
    },
  });
};

// Kakao 로그인
const signInWithKakao = async () => {
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'kakao',

      redirectTo: `${window.location.origin}/`,
    },
  });

```

### Email/Password 인증

```javascript
// 회원가입
const signUp = async (email, password, fullName) => {
  const { data, error } = await supabase.auth.signUp({
    email,
    password,
    options: {
      data: {
        full_name: fullName,
      },
    },
  });
};

// 로그인
const signIn = async (email, password) => {
  const { data, error } = await supabase.auth.signInWithPassword({
    email,
    password,
  });
};

// 비밀번호 재설정
const resetPassword = async (email) => {
  const { error } = await supabase.auth.resetPasswordForEmail(email, {
    redirectTo: `${window.location.origin}/reset-password`,
  });
};
```

## 데이터베이스 스키마

### User Profiles

```sql
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) PRIMARY KEY,
  full_name TEXT,
  display_name TEXT,
  avatar_url TEXT,
  api_keys JSONB DEFAULT '{}',
  signup_domain TEXT,
  visited_sites TEXT[] DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID | auth.users 외래키 (PK) |
| `full_name` | TEXT | 사용자 전체 이름 |
| `display_name` | TEXT | 표시 이름 |
| `avatar_url` | TEXT | 프로필 이미지 URL |
| `api_keys` | JSONB | 암호화된 API 키 저장 |
| `signup_domain` | TEXT | 가입 사이트 도메인 |
| `visited_sites` | TEXT[] | 방문한 사이트 목록 (cross-site tracking) |
| `created_at` | TIMESTAMPTZ | 생성일시 |
| `updated_at` | TIMESTAMPTZ | 수정일시 |

## Row-Level Security (RLS) 정책

### Profiles RLS

```sql
-- 자신의 프로필만 조회 가능
CREATE POLICY "Users can view own profile"
  ON profiles FOR SELECT
  USING (auth.uid() = id);

-- 자신의 프로필만 수정 가능
CREATE POLICY "Users can update own profile"
  ON profiles FOR UPDATE
  USING (auth.uid() = id);
```

### Teaching Tables RLS

```sql
-- 자신의 데이터만 CRUD 가능 (공통 패턴)
CREATE POLICY "Users can CRUD own data"
  ON syllabi FOR ALL
  USING (auth.uid() = user_id);

-- rubrics, assignments, feedback, evaluations도 동일 패턴
```

### Board RLS

```sql
-- 게시글: 모든 사용자 조회 가능, 작성자만 수정/삭제
CREATE POLICY "Anyone can view posts"
  ON posts FOR SELECT
  USING (true);

CREATE POLICY "Users can insert own posts"
  ON posts FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own posts"
  ON posts FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "Users can delete own posts"
  ON posts FOR DELETE
  USING (auth.uid() = user_id);

-- 댓글도 동일 패턴
```

## Cross-Site visited_sites 추적

OpenClaw와 Teaching AI는 동일한 Supabase 프로젝트를 사용하므로, 사용자가 어떤 사이트를 방문했는지 추적한다.

```javascript
// 사이트 방문 기록 업데이트
const updateVisitedSites = async (userId, siteDomain) => {
  const { data: profile } = await supabase
    .from('profiles')
    .select('visited_sites')
    .eq('id', userId)
    .single();

  const sites = profile?.visited_sites || [];
  if (!sites.includes(siteDomain)) {
    await supabase
      .from('profiles')
      .update({
        visited_sites: [...sites, siteDomain],
      })
      .eq('id', userId);
  }
};

// 로그인 시 호출
// updateVisitedSites(user.id, 'openclaw.dreamitbiz.com');
```
