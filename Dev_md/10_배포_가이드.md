| 항목 | 값 |

| **프리뷰** | Vercel |

| **출력 디렉토리** | `dist/` |

## 개발 서버

### 실행

```bash
npm run dev
```

- **포트**: 5175
- **URL**: http://localhost:5175
- **HMR**: 활성화 (Vite 기본)

### 환경 변수

`.env` 파일에 다음 변수를 설정한다:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

| 변수 | 설명 |
|------|------|
| `VITE_SUPABASE_URL` | Supabase 프로젝트 URL |
| `VITE_SUPABASE_ANON_KEY` | Supabase Anonymous Key (공개 키) |

> `VITE_` 접두사가 있어야 클라이언트 사이드에서 접근 가능하다.

## GitHub Pages 배포

### GitHub Actions 워크플로우

`.github/workflows/deploy.yml` 파일로 자동 배포를 설정한다.

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          VITE_SUPABASE_URL: ${{ secrets.VITE_SUPABASE_URL }}
          VITE_SUPABASE_ANON_KEY: ${{ secrets.VITE_SUPABASE_ANON_KEY }}

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### GitHub Secrets 설정

GitHub 저장소 Settings > Secrets and variables > Actions에 다음을 추가:

| Secret | 값 |
|--------|-----|
| `VITE_SUPABASE_URL` | Supabase 프로젝트 URL |
| `VITE_SUPABASE_ANON_KEY` | Supabase Anonymous Key |

## SPA 리다이렉트 핸들러

GitHub Pages는 SPA를 네이티브로 지원하지 않으므로, `public/404.html` 파일을 사용하여 모든 경로를 `index.html`로 리다이렉트한다.

### `public/404.html`

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>OpenClaw</title>
  <script type="text/javascript">
    // GitHub Pages SPA redirect
    // https://github.com/rafgraph/spa-github-pages
    var pathSegmentsToKeep = 0;
    var l = window.location;
    l.replace(
      l.protocol + '//' + l.hostname + (l.port ? ':' + l.port : '') +
      l.pathname.split('/').slice(0, 1 + pathSegmentsToKeep).join('/') + '/?/' +
      l.pathname.slice(1).split('/').slice(pathSegmentsToKeep).join('/').replace(/&/g, '~and~') +
      (l.search ? '&' + l.search.slice(1).replace(/&/g, '~and~') : '') +
      l.hash
    );
  </script>
</head>
<body></body>
</html>
```

### `index.html`에 리다이렉트 스크립트 추가

```html
<script type="text/javascript">
  // GitHub Pages SPA redirect handler
  (function(l) {
    if (l.search[1] === '/') {
      var decoded = l.search.slice(1).split('&').map(function(s) {
        return s.replace(/~and~/g, '&');
      }).join('?');
      window.history.replaceState(null, null,
        l.pathname.slice(0, -1) + decoded + l.hash
      );
    }
  }(window.location));
</script>
```

## .nojekyll

`public/.nojekyll` 파일 (빈 파일)을 추가하여 GitHub Pages의 Jekyll 처리를 비활성화한다. 이렇게 하면 `_`로 시작하는 파일/폴더도 정상적으로 서빙된다.

```bash
touch public/.nojekyll
```
